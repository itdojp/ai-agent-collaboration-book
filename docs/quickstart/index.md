# Quick Start: 90分で本書の型を使う

このページは、書籍全体を読む前に、実務で最小限の成果を出すための導線である。

## 目的

AIエージェント協働を「良さそうなプロンプト」ではなく、次の5点で運用する。

1. 仕事を選ぶ
2. 依頼を契約にする
3. コンテキストを渡す
4. 出力をレビューする
5. 次回改善に残す

## 90分の実行手順

### Step 1: 対象業務を1つ選ぶ（10分）

次の条件を満たす業務を選ぶ。

- 毎週または毎月発生する
- 情報整理、下書き、比較、分類、レビュー補助のいずれかを含む
- 失敗しても即座に顧客・本番・法務へ影響しない
- 成果物の良し悪しを人間がレビューできる

避けるべき対象は、契約判断、人事評価、価格決定、本番変更、顧客への自動送信、個人情報を含む処理である。

### Step 2: AIエージェント依頼書を書く（20分）

付録Aの「AIエージェント依頼書」を使う。最低限、次を埋める。

```text
目的：
最終成果物：
対象範囲：
除外範囲：
入力情報：
制約条件：
出力形式：
品質基準：
人間承認が必要な条件：
停止条件：
```

### Step 3: Context Packを作る（20分）

AIに渡す情報を、資料の丸投げではなく、次の形式に圧縮する。

```text
業務背景：
現在の状況：
過去の経緯：
決定済み事項：
未決事項：
重要な制約：
情報の鮮度：
確定情報：
推定情報：
要確認事項：
```

### Step 4: AIに実行させる（15分）

依頼書とContext Packを入力し、次を明示する。

```text
まず前提の不足を指摘してください。
不足があれば、成果物作成に入る前に質問してください。
不明点は推測せず「要確認」に分類してください。
```

### Step 5: AI出力レビュー表で判定する（15分）

付録Aの「AI出力レビュー表」を使い、少なくとも次を判定する。

- 目的適合性
- 事実正確性
- 根拠
- 論理
- リスク
- 不確実性の分離
- 最終判断

### Step 6: 改善ログを残す（10分）

次回改善のため、Agent Run Logに次を残す。

```text
うまくいった指示：
不足したコンテキスト：
レビューで見つかった問題：
次回修正するテンプレート：
```

## 90分後の成果物

- AIエージェント依頼書 1件
- Context Pack 1件
- AI出力レビュー表 1件
- 改善ログ 1件

## 3時間版: 6層モデルで拡張する

90分版は、低リスクな個人・チーム利用の最小導線である。顧客影響、社内データ、ツール実行、複数人レビューを含む場合は、次の3時間版を使う。

3時間版は、横長の表ではなく、時間帯ごとの小さな実行単位として進める。

1. **0:00-0:20 / Work**
   - 実施内容: 対象業務を選び、AI適用判断を行う。
   - 成果物: AI適用判断シート。
2. **0:20-0:45 / Request**
   - 実施内容: 目的、範囲、制約、成果物、受け入れ条件を定義する。
   - 成果物: Request Contract。
3. **0:45-1:10 / Context**
   - 実施内容: 前提、鮮度、確度、機密区分を整理する。
   - 成果物: Context Pack。
4. **1:10-1:35 / Delegation**
   - 実施内容: 作業を分解し、チェックポイントと停止条件を置く。
   - 成果物: Task Brief / Stop Rules。
5. **1:35-2:00 / Control**
   - 実施内容: 出力レビュー、権限、HITL、ログを設計する。
   - 成果物: AI出力レビュー表 / HITL承認フロー / Agent Run Log。
6. **2:00-2:25 / Security**
   - 実施内容: 入力禁止情報、外部入力、ログ、ツール権限を確認する。
   - 成果物: AI Security Checklist。
7. **2:25-2:45 / Human / Org**
   - 実施内容: 採用・棄却・実験化・停止の判断を定義する。
   - 成果物: Decision Brief / AI実験設計シート。
8. **2:45-3:00 / Improvement**
   - 実施内容: 次回改善とOwnerを決める。
   - 成果物: Improvement Backlog。

3時間版の完了条件は、AI出力を1つ作ることではない。次の問いに答えられる状態である。

```text
どの仕事にAIを使うか。
何をAIに依頼するか。
どの前提を渡すか。
どこまで委任するか。
どこで止めるか。
誰が何を承認するか。
どのログを残すか。
どのリスクが残るか。
次に何を改善するか。
```
