# Recurring Case: 法人向けバックアップサービスのAI活用

本書全体で繰り返し使う架空ケースである。章ごとの説明を断片化させないため、同じ業務文脈で成果物を積み上げる。

## 架空企業

```text
会社名：ITDO Cloud Services（架空）
事業：法人向けバックアップ、監視、マネージドインフラ支援
主な顧客：中堅企業、SaaS企業、情報システム部門
```

## Recurring business scenarios

| シナリオ | 使う章 | 内容 |
|---|---|---|
| 競合調査 | 第2〜6章 | 法人向けバックアップ市場の競合比較を作る |
| 顧客問い合わせ対応 | 第1章、第7章 | FAQ検索から回答案作成、人間承認までを設計する |
| 障害報告 | 第4章、第5章、第8章 | ログと時系列から報告書を作る |
| 価格改定判断 | 第11章 | AI分析を参考にしつつ、人間が判断する |
| 営業と開発の対立 | 第12章 | 要望優先度をめぐる対立を処理する |
| 新提案実験 | 第13章 | AIを使って小さな仮説検証を回す |
| 社内展開 | 第15〜16章 | AI活用を職種別研修へ落とす |

## このケースで守るルール

- 実在企業の非公開情報は使わない。
- 価格、仕様、法令など変化しやすい情報は、本文では架空データまたは要確認扱いにする。
- 顧客影響がある判断はAIに確定させない。
- AIによる外部送信、本番変更、契約判断は必ず人間承認を挟む。
