# Knowledge Architecture 統合計画

## 方針

本書は案A、すなわち新刊追加とする。既存の『生成AIコミュニケーション技術』は置換しない。ただし、将来的な統合・改題候補として位置づける。

## 推奨カテゴリ

```text
AI Agent Enablement（AIエージェント活用・組織展開）
```

## 追加位置

既存の書籍一覧では、以下のどちらかが候補である。

### 案1: Professional Foundations の末尾に置く

AI活用を全社員の基礎リテラシーとして扱う場合に適する。

```text
Professional Foundations
1. 根拠で進める開発仕事術
2. チケット駆動の仕事術
3. エンジニアリングドキュメント実践ガイド
4. セキュリティ＆プライバシー基礎リテラシー
5. インシデント対応 基礎
6. AIエージェント協働の仕事術
```

### 案2: 新カテゴリを作る

既存シリーズのAI関連書籍が増えているため、独立カテゴリにする方が整理しやすい。

```text
AI Agent Enablement
1. AIエージェント協働の仕事術
2. AIエージェント・コミュニケーション実践ガイド
3. AIエージェント実践: Prompt / Context / Harness Engineering
4. GitHub AgentOps 実践ガイド
5. AI主導開発時代のソフトウェアテスト戦略
6. AI時代のプロフェッショナルITエンジニアの思考法
```

推奨は案2である。理由は、AIエージェント協働は Professional Foundations の一部であると同時に、開発・運用プロセス、ソフトスキル、組織展開を横断するためである。
