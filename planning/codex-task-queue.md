# Codex Task Queue

Codex CLIで進める具体タスクを、順番に実行できる粒度へ分解する。

## Queue 1: リポジトリ整備

```text
このリポジトリをIT Engineer Knowledge Architectureシリーズの新規書籍として整備してください。book-config.json、index.md、README.md、quickstart.md、concept-map.md、artifact-index.md、troubleshooting.md、figure-index.md のリンク整合性を確認し、欠けているナビゲーションを補ってください。
```

## Queue 2: 第1〜4章の初稿品質向上

```text
manuscript/ch01〜ch04を公開可能な初稿に拡張してください。各章に、導入、典型失敗、設計原則、架空ケース、作成成果物、章の受け入れ基準、演習を含めてください。概念説明だけで終えず、付録テンプレートへ必ず接続してください。
```

## Queue 3: 第5〜10章の運用・統制強化

```text
manuscript/ch05〜ch10を、AIエージェント運用に必要な委任、レビュー、ツール権限、ログ、セキュリティ、ガバナンスの観点で拡張してください。各章で「事故が起きる条件」「安全に始める最小構成」「監査・改善に残すログ」を追加してください。
```

## Queue 4: 第11〜14章の人間側能力を業務設計として再記述

```text
manuscript/ch11〜ch14を、心理論ではなく業務上の意思決定・対話・実験・行動停止の設計として書き直してください。discernment、connection debt、pace/spin、self-talkは、必ずテンプレート、チェックリスト、演習へ落としてください。
```

## Queue 5: 第15〜16章と付録を研修化へ接続

```text
manuscript/ch15〜ch16とappendicesを、社内教材化・認定・KPI測定に使える粒度へ拡張してください。ロール別学習パス、実技試験、評価ルーブリック、MVP教材20点、90日展開計画を相互リンクしてください。
```

## Queue 6: Source Notesと本文の整合

```text
source-notes.mdの出典候補を確認し、本文で外部フレームワークやツール仕様に触れる箇所に出典メモを追加してください。変化しやすい情報は断定しすぎず、公式情報確認を促す表現へ修正してください。
```

## Queue 7: 最終レビュー

```text
planning/review-checklist.mdとplanning/chapter-acceptance-criteria.mdに基づき、全章・付録・ナビゲーションをレビューしてください。重複、章間の矛盾、未定義用語、リンク切れ、精神論に寄りすぎた表現、法務・セキュリティの断定しすぎを修正してください。
```
