# Review Checklist

## シリーズUXチェック

- [ ] ヘッダーに書籍タイトルとトップ導線がある
- [ ] サイドバーまたは同等のナビゲーションに章・付録への導線がある
- [ ] フッターにライセンスとリポジトリ導線がある
- [ ] `book-config.json` に `ux.profile` と `ux.modules` が定義されている
- [ ] Profile B の必須 modules: `checklistPack`, `troubleshootingFlow`, `figureIndex` が有効である

## 内容チェック

- [ ] 本書の対象読者が明確である
- [ ] 本書で扱うこと、扱わないことが明確である
- [ ] 既存書籍との責務境界が明確である
- [ ] 各章に学習目標がある
- [ ] 各章に典型的な失敗がある
- [ ] 各章に実務ケースがある
- [ ] 各章に成果物テンプレートへの接続がある
- [ ] 各章に演習がある

## ファクト・出典チェック

- [ ] 変化しやすいツール仕様は公式情報への確認を促している
- [ ] 標準、法規制、セキュリティフレームワークは一次情報を参照している
- [ ] Lenny's Newsletter 由来の内容は、実践仮説として扱っている
- [ ] 過度な引用がない

## リスクチェック

- [ ] AIに丸投げしてはいけない業務が明示されている
- [ ] 顧客送信、本番変更、契約、採用、人事評価、支払いなどの高リスク操作で人間承認を求めている
- [ ] 機密情報、個人情報、認証情報の入力禁止または制限が明示されている
- [ ] プロンプトインジェクション、権限過大、外部ツールリスクが扱われている
- [ ] AI出力をそのままコード、SQL、メール、契約判断に使わない旨が明確である

## 教材化チェック

- [ ] 全社員向け教材へ切り出せる章がある
- [ ] 管理職向け教材へ切り出せる章がある
- [ ] エンジニア向け教材へ切り出せる章がある
- [ ] テンプレートがコピーして使える粒度になっている
- [ ] スキル評価ルーブリックがある
